  • Performance : The script is lightweight to reduce high resource consumption from solutions that utilize external libraries such as JQuery to run.

3.2 Alternative CSS

Mobile CSS

This includes a mobile adaptation of my website's main styles. I am utilizing media queries to make the following changes to make the site mobile friendly. First, given my gallery is a terrible experience on a smaller screen when the screen reaches the point where it's unusable, I hide the gallery and replace it with a static image. Second, given screen width is rather narrow on mobile, all my content (text and tiles) stretch to be wider to make it easier to read. Third, the social media feeds are not condusive of a good mobile experience, given you have such little screen real estate there should be only one thing to focus on so as we reach mobile screen sizes they also vanish. Fourth, Given it's a flexbox based website, I have been able to convert all of the elements in each pane from a row layout on desktop to a column layout on mobile. It allows for a much nicer experience as mobile screens are not wide enough to display rows of tiles.

3.4 Social Networking Feed(s)

At the bottom right of the screen, click the twitter bubble

Notes:

  • Very proud of this one. I have developed my very own feed, including a hovering button, a back-end server utilizing REST API calls. It works very similarly to a chat window on a service like Facebook Messenger.
  • I developed a NodeJS based backend server that stores a token and interacts with the Twitter API. It is hosted on a free Amazon Web Services host. Note : I do understand the back end is not assessable, but if you want to look at the code I wrote please contact me.
  • My front end website utilizing this script requests my API to request the Twitter API for my tweets. Twitter API responds to my API which then responds to my site.
  • After getting the API response (which is in the form of a JSON Array File), my script handles the population of the data onto my DOM. Note: the script is heavily commented given it is slightly outside of the scope of this course
  • Additionally, I have set this up with Asynchronous Javascript, so, until I actually get a response from the server (if not I use the backup data), the buttons to access the twitter feed will not display. If both data sources fail, the user will not see the button and wonder why it's unresponsive.
  • Note: I did ask for special permission to do something very similar to this from Sabrina she did say yes, but I just extended it a bit with NodeJS API and using Twitter as my source instead.
