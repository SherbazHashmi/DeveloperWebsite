Shout Out Process

Shout Out's a project of mine which won ESRI's Young Scholar of the Year award. It is a community based reporting platform which intends to revolutionise the current reporting process. It's a platform that provides a wholistic solution with multiple phases.

The diagram below shows the phases of the solution, with each phase having their own web application.

Select any phase on the diagram below to understand.

Report

Community Reporting is done through a single Survey123 form which serves all members of every state that utilizes the service. The community users are prompted to provide their details and summary of the incident with a photograph. The form is accessible through a platform agnostic web application as well as through a native mobile application.

Moderation

In order to validate the reports, the community reports are all collated and displayed in an ESRI Web Application with an interactive map. The moderation team evaluates each incident and verifies it by contacting the reporter directly. Upon contact, any extra details are noted down and the report is marked as verified in the system.

The Moderation Web Application allows the moderators to modify the incident's details, location and contact information of the reporter.

Prioritisation

After the reports have been moderated, they are allocated to the relevant state. Each state has its own unique dashboard. The dashboards contain all the incidents on an interactive web map for the relevant state, with real time statistics based on the region of the map being observed

Task Allocation

The tasks are finally allocated to the departments utilizing ESRI Workforce which is able to assign tasks to each department. It is assumed that each individual department already has a task management system, hence each member of the department is not assigned tasking through ESRI Workforce.

However, in the case that there is no pre-existing task management system, ESRI Workforce would be able to assign each issue with data to each individual in the department. Note: Member specific dashboards are not implemented as of writing.
